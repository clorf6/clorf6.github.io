T2 动物园

解析：这 TM 才是真正的签到题，枯了，CCF 居然开始不按难度梯度来排题了。

一道很水的题，对于每 111 位，如果动物园中有这一位为 111 的动物或者这 111 位不需要购买任何饲料，那么这 111 位对饲料就没有影响，即新的动物这 111 位可以为 111。假设这样的位有 ppp 个，答案即为 2p−n2 ^{p}-n2p−n。唯一需要注意的地方是数据范围中，当 n=0,p=64n=0,p=64n=0,p=64 时，答案会爆 unsigned long long，因此需要特判一手。(考场上居然又没特判，我就这)

#include<cstdio>
#include<cstring>
#include<cstdlib>
#include<cmath>
#include<algorithm>
#include<bitset>
#include<vector>
using namespace std;
#define INF 1e9
const int maxn=1000010;
bitset<100> p;
vector<int> a[110];
int n,m,c,k;
unsigned long long ans,cnt;
int main()
{
	scanf("%d%d%d%d",&n,&m,&c,&k);
	for(int i=1;i<=n;i++)
	{
		unsigned long long x;
		scanf("%llu",&x);
		p|=x;
	}
	for(int i=1;i<=m;i++)
	{
		int x,y;
		scanf("%d%d",&x,&y);
		a[x].push_back(y);
	}
	for(int i=0;i<k;i++)
	{
		long long r=a[i].size();
		if(r<=0||p[i])
			cnt++;
	}
	if(n==0&&cnt==64)
	{
		printf("18446744073709551616");
		return 0;
	}
	if(cnt==0)
		ans=(1ull<<cnt)-n;
	else
		ans=(1ull<<(cnt-1))-n+(1ull<<(cnt-1));
	printf("%llu",ans);
	return 0;
}

T3 函数调用

解析：一道比较正常的题。其实想法并不难，但是就是没有想出来。

我们可以把题目看成把所有数都乘上一个数 mulmulmul，然后再给其中一些数加上 addiadd _{i}addi​，其中 mulmulmul 是所有被调用的第二类函数给所有数乘的 VjV _{j}Vj​ 的积。

函数的调用关系是 DAG，这个不难发现，因此可以通过建图后，一遍 dfs 求出 mulimul _{i}muli​ 代表第 iii 个函数调用后能将所有数乘上 mulimul _{i}muli​。

所以现在的难点在于如何求 addiadd _{i}addi​。

我们假设其中一个数的操作序列(把第三类操作展开)是 abaabbaab....\texttt{abaabbaab....}abaabbaab....，其中 a\texttt{a}a 代表第一个函数操作，加上某个数 kkk，bbb 代表第二个函数操作，乘上某个数 ppp，那么这个数的答案为：

ansi={[(ai+k1)p1+k2]p2+k3}p3+⋯=(p1p2p3⋯ )ai+(p1p2p3⋯ )k1+(p2p3⋯ )k2+(p3⋯ )k3+⋯\begin{aligned} ans _{i} & =\{[(a _{i}+k _{1})p _{1}+k _{2}]p _{2}+k _{3} \}p _{3}+\cdots \\ & =(p _{1}p _{2}p _{3}\cdots)a _{i}+(p _{1}p _{2}p _{3}\cdots)k _{1}+(p _{2}p _{3}\cdots)k _{2}+(p _{3}\cdots)k _{3}+\cdots \end{aligned} ansi​​={[(ai​+k1​)p1​+k2​]p2​+k3​}p3​+⋯=(p1​p2​p3​⋯)ai​+(p1​p2​p3​⋯)k1​+(p2​p3​⋯)k2​+(p3​⋯)k3​+⋯​

拆开后，我们发现每个第一类操作加的数都会最终乘上一个系数，这个数就是在他之后的所有第二类操作和第三类操作乘的数的积。我们只要把这个系数求出来即可。我们通过算每个操作对它所调用的单点加法的贡献来计算这个系数。

我们设 dpidp _{i}dpi​ 代表第 iii 个函数对它所调用的单点加法产生了 dpidp _{i}dpi​ 倍的贡献，我们可以先把每个函数一开始调用时的产生的贡献求出，对于第三类函数先不处理，最后再用拓扑排序更新真正的 dpidp _{i}dpi​。一开始我们可以倒序来处理所有函数调用，同时算出 mulmulmul 来：

  • 对于第一类函数，我们令 dpi←dpi+muldp _{i}\gets dp _{i}+muldpi​←dpi​+mul；
  • 对于第二类函数，我们令 mul←mul×Vimul\gets mul\times V _{i}mul←mul×Vi​；
  • 对于第三类函数，我们令 dpi←dpi+mul,mul←mul×mulidp _{i}\gets dp _{i}+mul,mul\gets mul\times mul _{i}dpi​←dpi​+mul,mul←mul×muli​。

然后我们再拓扑排序，求出每个第三类操作真正的 dpidp _{i}dpi​，同时算出 addiadd _{i}addi​。

  • 对于第一类函数，我们令 addPi←addPi+fi×Viadd _{P _{i}}\gets add _{P _{i}}+f _{i}\times V _{i}addPi​​←addPi​​+fi​×Vi​；
  • 对于第三类函数，我们从后往前处理所有调用的函数。我们令 w=dpiw=dp _{i}w=dpi​，然后对于每个调用的函数 jjj，令 fj←fj+fif _{j}\gets f _{j}+f _{i}fj​←fj​+fi​，然后使 w←w×muljw\gets w\times mul _{j}w←w×mulj​。

最后每个数输出 ai×mul+addia _{i}\times mul+add _{i}ai​×mul+addi​ 即可。时间复杂度 O(n+m)O(n+m)O(n+m)。

T4 贪吃蛇

解析：一道看起来很简单，但是有很多细节的题。

首先有一个结论：当前最强的蛇吃了最弱的蛇后，如果没有变成最弱的蛇，那么他一定选择从吃。

证明：假如当前蛇从小到大排序为 a1,a2,⋯ ,ana _{1},a _{2},\cdots,a _{n}a1​,a2​,⋯,an​，所以最强的为 ana _{n}an​，最弱的为 a1a _{1}a1​，吃掉后会变成 an−a1a _{n}-a _{1}an​−a1​。现在我们分类讨论：

  • 假如 an−a1a _{n}-a _{1}an​−a1​ 依然是最强的，那么肯定吃；
  • 假如 an−a1a _{n}-a _{1}an​−a1​ 不是最强的，那么此时 an−1a _{n-1}an−1​ 是最强的，a2a _{2}a2​ 是最弱的，吃掉后变成 an−1−a2a _{n-1}-a _{2}an−1​−a2​，又因为 a1≤a2,an−1≤ana _{1}\le a _{2},a _{n-1}\le a_{n}a1​≤a2​,an−1​≤an​，所以 an−1−a2≤an−a1a _{n-1}-a _{2}\le a_{n}-a _{1}an−1​−a2​≤an​−a1​。即后面的蛇吃掉后会比 an−a1a _{n}-a _{1}an​−a1​ 更弱。因此如果后面的蛇不会死掉，那么我这条 an−a1a _{n}-a _{1}an​−a1​ 的蛇也不会死。因为有蛇会死在我前面，所以可以吃。

有了这个结论，我们可以大胆地吃一部分蛇。但是如果吃了变成最弱地蛇我们到底还要吃不吃呢？

可以来推一下：

如果 ana _{n}an​ 吃掉 a1a _{1}a1​ 后变成了最弱的蛇，那么下次该用 an−1a _{n-1}an−1​ 吃 an−a1a _{n}-a _{1}an​−a1​ 这条蛇，如果 an−1−(an−a1)a _{n-1}-(a _{n}-a _{1})an−1​−(an​−a1​) 不是最弱的蛇，那么他会选择吃，这样 ana _{n}an​ 就会凉掉，所以 ana _{n}an​ 当初选择不吃。如果 an−1−(an−a1)a _{n-1}-(a _{n}-a _{1})an−1​−(an​−a1​) 是最弱的蛇，那么就要考虑下一条最强蛇 an−2a _{n-2}an−2​ 吃掉这条 an−1−(an−a1)a _{n-1}-(a _{n}-a _{1})an−1​−(an​−a1​) 的蛇后是否是最弱的蛇了。如果吃掉后不是最弱的蛇就会选择吃，这样 an−1a _{n-1}an−1​ 就凉了，所以 an−1a _{n-1}an−1​ 当初不会吃，所以 ana _{n}an​ 就不会死，所以可以吃。如果吃了变成了最弱的蛇，就会考虑下一条蛇……

这个问题就是一种递归了，直到某条蛇吃了后不是最弱的蛇或者只剩下 222 条蛇为止。这样最后一条蛇会吃，倒数第二条为了保命不吃，倒数第三条可以吃，倒数第四条会不吃，倒数第五条可以吃……

这样，ana _{n}an​ 吃不是就和最后一条蛇的奇偶性相关了。如果 ana _{n}an​ 不吃游戏就结束，ana _{n}an​ 吃掉后，那么 an−1a _{n-1}an−1​ 必定不吃，下一轮结束。

所以我们模拟两个阶段即可：

  • 阶段一：所有最强🐍吃掉后都不是最弱的，都会选择吃；
  • 阶段二：所有最强🐍吃掉后都是最弱🐍，直到有条蛇吃了之后不是最弱或者只剩两条为止。

阶段一结束后，我们就直接判断阶段二的奇偶性看看能不能再吃一次即可。我们用 set 可以方便维护最强最弱蛇，时间复杂度 O(Tnlog⁡n)O(Tn\log n)O(Tnlogn)，期望得分 707070 分。

如果我们采用 [NOIP2016] 蚯蚓里的做法，用两个双端队列 q1,q2q _{1},q _{2}q1​,q2​ 来维护单调性，那么时间复杂度为 O(Tn)O(Tn)O(Tn)，期望得分 100100100 分。

所以这样的做法为：

  • 阶段一：每次从 q1,q2q _{1},q _{2}q1​,q2​ 尾部取出最强的蛇，从 q1q _{1}q1​ 头部取出最弱的蛇，如果吃了是最弱的蛇，那么就进入第二阶段，否则直接装入 q2q _{2}q2​ 的头部；
  • 阶段二：此时最弱的蛇一直是吃蛇的那条蛇，所以没必要丢队列里，单独维护一下即可。

int t,n,a[maxn],k,ans;
deque<pair<int,int> > q1,q2;
int main()
{
	scanf("%d",&t);
	for(int _=1;_<=t;_++)
	{
		if(_==1)
		{
			scanf("%d",&n);
			for(int i=1;i<=n;i++)
				scanf("%d",&a[i]);
		}
		for(int i=1;i<=n;i++)
			q1.push_back(make_pair(a[i],i));
		while(1)
		{
			if(q1.size()+q2.size()==2)
			{
				ans=1;
				break;
			}//只剩2条蛇
			int x,y,id;
			y=q1.front().first;
			q1.pop_front();//最弱
			if(q2.empty()||(!q1.empty()&&q1.back()>q2.back()))
			{
				x=q1.back().first;
				id=q1.back().second;
				q1.pop_back();
			}
			else
			{
				x=q2.back().first;
				id=q2.back().second;
				q2.pop_back();
			}//最强
			pair<int,int> z=make_pair(x-y,id);//吃掉后
			if(q1.empty()||q1.front()>z)//成为最弱，进入第二阶段
			{
				ans=q1.size()+q2.size()+2;//这一步不吃剩下的蛇
				int flag=0;
				while(1)
				{
					flag++;
					if(q1.size()+q2.size()+1==2)
					{
						if(!(flag&1))
							ans--;
						break;
					}
					int x,id;
					if(q2.empty()||(!q1.empty()&&q1.back()>q2.back()))
					{
						x=q1.back().first;
						id=q1.back().second;
						q1.pop_back();
					}
					else
					{
						x=q2.back().first;
						id=q2.back().second;
						q2.pop_back();
					}//最强
					z=make_pair(x-z.first,id);//更新最弱
					if(!((q1.empty()||z<q1.front())&&(q2.empty()||z<q2.front())))//不是最弱
					{
						if(!(flag&1))
							ans--;
						break;
					}
				}
				break;
			}
			else
				q2.push_front(z);
		}
		printf("%d\n",ans);
		if(_<t)
		{
			scanf("%d",&k);
			for(int i=1;i<=k;i++)
			{
				int x,y;
				scanf("%d%d",&x,&y);
				a[x]=y;
			}
			while(!q1.empty())
				q1.pop_back();
			while(!q2.empty())
				q2.pop_back();
		}
	}
	return 0;
}
